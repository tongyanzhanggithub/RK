# RepairKit Supply 上线部署指南

把整个应用部署到 **Vercel + 云数据库（Neon PostgreSQL）+ Vercel Blob 图片存储**，并完成后台安全加固。
按顺序照做即可，标注 👉 的是需要你亲自操作的步骤。

---

## 总览：你需要注册的账号

| 服务 | 用途 | 费用 |
|------|------|------|
| [Vercel](https://vercel.com) | 托管整个网站 | 免费档够用 |
| [Neon](https://neon.tech) | 云 PostgreSQL 数据库 | 免费档够用 |
| [Stripe](https://stripe.com) | 收款（你已有测试账号） | 按成交抽成 |
| 域名商（阿里云/Namecheap/Cloudflare） | 买正式域名 | 约 ¥60–100/年 |
| SMTP 邮件服务（可选） | 发订单/批发邮件 | 多数有免费额度 |

## 第 1 步：准备云数据库（Neon）

👉 1. 打开 https://neon.tech，用 GitHub 账号登录
👉 2. 点 **Create Project**，区域选离客户近的（如 Singapore），项目名随意
👉 3. 创建后，在 **Connection string** 里复制连接串，形如：
```
postgresql://user:password@ep-xxx.ap-southeast-1.aws.neon.tech/dbname?sslmode=require
```
把它先存到记事本，下面要用。

> 说明：本项目开发期用的是 SQLite（本地文件），上线必须换成 PostgreSQL。
> 下一步会把代码切换过去。

---

## 第 2 步：把代码从 SQLite 切到 PostgreSQL

> 这一步会改一行代码并重建数据库结构。改完后，**本地开发也要连 Neon**
> （或 Neon 的另一个分支），不再用本地 SQLite 文件。

👉 1. 打开 `prisma/schema.prisma`，把最上面的：
```prisma
datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}
```
改成：
```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

👉 2. 在本地 `.env` 里把 `DATABASE_URL` 改成第 1 步复制的 Neon 连接串。

👉 3. 在项目目录运行（在 Neon 上建好所有表）：
```powershell
npx prisma generate
npx prisma db push
```

👉 4. 创建管理员账号（换成你自己的邮箱和强密码）：
```powershell
$env:ADMIN_EMAIL='你的邮箱'; $env:ADMIN_PASSWORD='一个强密码'; npm run admin:create
```

> 注意：`prisma/seed.js` 和 `scripts/migrate-*.js` 是给旧 SQLite 用的，
> 切到 PostgreSQL 后**不要再运行它们**。表结构由 `prisma db push` 创建，
> 产品数据直接在后台手动录入即可。

## 第 3 步：把代码推到 GitHub

确认所有改动已提交并推送（你的仓库：https://github.com/tongyanzhanggithub/RK ）：

git add -A

## 第 4 步：在 Vercel 部署

👉 1. 打开 https://vercel.com，用 GitHub 登录
👉 2. 点 **Add New → Project**，选择 `RK` 仓库，点 **Import**
👉 3. 在 **Environment Variables** 里逐条填入（参考项目里的 `.env.production.example`）：

| 变量名 | 值 |
|--------|-----|
| `DATABASE_URL` | Neon 连接串 |
| `NEXT_PUBLIC_SITE_URL` | 先填 Vercel 给的临时域名，绑定域名后再改 |
| `NEXT_PUBLIC_WHATSAPP_NUMBER` | 你的 WhatsApp 号（纯数字含国家码） |
| `NEXT_PUBLIC_CONTACT_EMAIL` | 你的客服邮箱 |
| `STRIPE_SECRET_KEY` | Stripe **Live** 模式密钥（第 6 步获取） |
| `STRIPE_WEBHOOK_SECRET` | Stripe Webhook 密钥（第 6 步获取） |
| `ADMIN_SESSION_SECRET` | 一段长随机串（见下方生成方法） |
| `SMTP_*` 系列 | 邮件配置（可选，先不填邮件功能就不启用） |

> 生成 `ADMIN_SESSION_SECRET`：在本地运行
> `node -e "console.log(require('crypto').randomBytes(48).toString('base64url'))"`
> 把输出整段复制进去。**这个值必须填，否则生产环境会拒绝启动。**

👉 4. 点 **Deploy**，等 1–2 分钟构建完成。

---

## 第 5 步：开通图片云存储（Vercel Blob）

> 上线后后台上传的产品图必须存到云存储，否则会丢失。

👉 1. 在 Vercel 项目页 → **Storage** 标签 → **Create Database** → 选 **Blob**
👉 2. 创建后它会自动把 `BLOB_READ_WRITE_TOKEN` 注入到项目环境变量
👉 3. 回到 **Deployments**，点最新部署的 **Redeploy** 让新变量生效

代码已经写好：检测到 `BLOB_READ_WRITE_TOKEN` 就自动用云存储，本地开发没有该变量则用本地文件夹，无需改代码。

---

## 第 6 步：配置 Stripe 正式收款

👉 1. 登录 https://dashboard.stripe.com ，右上角从 **Test mode** 切到 **Live mode**
👉 2. **Developers → API keys**，复制 **Secret key**（`sk_live_...`），填到 Vercel 的 `STRIPE_SECRET_KEY`
👉 3. **Developers → Webhooks → Add endpoint**：
   - Endpoint URL 填：`https://你的域名/api/stripe/webhook`
   - 选择事件：`checkout.session.completed`、`payment_intent.succeeded`、`payment_intent.payment_failed`、`charge.refunded`
👉 4. 创建后复制 **Signing secret**（`whsec_...`），填到 Vercel 的 `STRIPE_WEBHOOK_SECRET`
👉 5. 改完环境变量后再 **Redeploy** 一次

---

## 第 7 步：绑定正式域名

👉 1. Vercel 项目页 → **Settings → Domains → Add**，输入你买的域名
👉 2. 按 Vercel 提示，去域名商后台添加它给的 DNS 记录（A 记录或 CNAME）
👉 3. 等 DNS 生效（几分钟到几小时），Vercel 会自动签发 HTTPS 证书
👉 4. 把 Vercel 里的 `NEXT_PUBLIC_SITE_URL` 改成 `https://你的域名`，并更新 Stripe Webhook 的 URL，然后 Redeploy

## 第 8 步：后台安全加固（已内置 + 建议）

代码已经做好的加固：
- ✅ 强制 HTTPS（HSTS 响应头）
- ✅ 防点击劫持、防 MIME 嗅探、Referrer 收敛等安全响应头
- ✅ 后台页面禁止搜索引擎收录（`X-Robots-Tag: noindex`）
- ✅ 密码 PBKDF2 加密、登录失败锁定、会话指纹失效、密码找回流程
- ✅ 生产环境强制要求 `ADMIN_SESSION_SECRET`

建议你额外做：
1. **管理员强密码**：第 2 步创建账号时用 16 位以上随机密码
2. **后台 IP 白名单**（可选）：如果你固定在公司/家里管理，在 Vercel 加环境变量
   `ADMIN_IP_ALLOWLIST="你的公网IP"`（多个用逗号分隔），后台就只允许这些 IP 访问。
   查自己公网 IP：浏览器搜 "my ip"。注意家庭宽带 IP 会变，变了要更新。
3. **定期改密码**：通过后台的「忘记密码」流程或 `admin:create` 命令。

## ⚠️ 一个技术注意点（无服务器环境的限流）

登录失败锁定、密码重置限流目前存在服务器内存里。Vercel 是无服务器架构，
多个实例之间不共享内存，所以这些限流在高并发下可能不严格。
**对你现在的流量完全够用**，等订单量大了再升级成数据库/Redis 存储（届时我可以帮你改）。
