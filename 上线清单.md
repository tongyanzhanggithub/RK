## 阶段 3 · HTTPS（免费证书 + 自动续期）

服务器上（Nginx 已装）：

```bash
sudo apt update && sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d partavio.com -d www.partavio.com
```

按提示填邮箱、同意条款、选择「自动把 http 跳转到 https」。

- [ ] certbot 成功签发证书
- [ ] `https://partavio.com` 出现小绿锁
- [ ] `http://partavio.com` 自动跳转到 `https://`
- [ ] certbot 自动续期已就绪（`sudo certbot renew --dry-run` 通过）

### ⚠️ 上 HTTPS 后必须改一处环境变量
编辑服务器 `.env.production`，把站点地址改成 https 域名（登录 Cookie 的 secure 开关依赖它）：

```
NEXT_PUBLIC_SITE_URL="https://partavio.com"
```

然后重建并重启：

```bash
set -a; . ./.env.production; set +a
npm run build && pm2 reload repairkit
```

- [ ] `NEXT_PUBLIC_SITE_URL` 已改为 `https://partavio.com`
- [ ] 重建重启后，后台能正常登录（不再出现「点登录又跳回登录页」）

## 阶段 8 · 安全运维（强烈建议，关系到"会不会被打崩"）

> 代码层已内置：SQL 全参数化、后台会话 HMAC 签名 + pbkdf2、登录爆破限流、支付 Webhook 验签、上传白名单+无路径穿越、安全响应头（HSTS/防点击劫持/nosniff）、JSON-LD 转义。下面是**运维侧**必须你来落实的：

- [ ] **HTTPS 必须开**（阶段 3 的 certbot）——否则后台登录 Cookie 明文传输，能被中间人窃取。上完 HTTPS 记得把 `NEXT_PUBLIC_SITE_URL` 改成 `https://…` 并重建（Cookie 的 Secure 开关依赖它）。
- [ ] **服务器防火墙只开必要端口**：阿里云安全组**只放行 80 / 443 / 22(SSH)**；
      **数据库 5432 绝不能对公网开放**（只允许本机 `localhost` 连，自建 PG 默认即是，确认安全组没放 5432）。
- [ ] **生成强随机密钥**（每个都不同），填进 `.env.production`：
  ```bash
  # 在服务器上各跑一次，把输出分别填给 ADMIN_SESSION_SECRET / CUSTOMER_SESSION_SECRET / CRON_SECRET
  openssl rand -base64 48
  ```
- [ ] **后台只许自己的 IP 进**（可选但很有效）：`.env.production` 加 `ADMIN_IP_ALLOWLIST="你的公网IP"`（多个逗号分隔），重建重启。
- [ ] **SSH 加固**：禁用 root 密码登录、改用密钥；非默认端口更佳。
- [ ] **数据库每日备份**并验证能恢复（`scripts/rk-backup.sh` 配 crontab，备份文件存到对象存储/异地）。
- [ ] **定期更新**：每隔一阵 `git pull && npm install && npm audit`，关注 Next/依赖安全更新（Next 大版本升级要单独测试，别盲升）。
  - 当前 `npm audit` 有 3 个漏洞,但修复都需**大版本跳级**,**暂不盲升**(待单独测试)：
    - `next` (RSC 缓存投毒 / 中间件绕过) + 其依赖的 `postcss` XSS → 修复要 `next@16`(14→16 破坏性升级,需整站回归测试)。其中"中间件绕过"仅影响 **Pages Router + i18n**,本站是 App Router,基本不适用。
    - `nodemailer` 高危,但仅在使用 `raw` 报文选项时触发,本站发信未用该选项,影响有限;可单独 `npm i nodemailer@latest` 验证后再升。
- [ ] 后台测试账号删除、`ADMIN_PASSWORD` 用 12 位以上强密码。

> 一句话：**HTTPS + 防火墙只开 80/443/22 + 强随机密钥 + 每日备份**，这四样做到，被轻易打崩的概率极低。

## 通用：改完 `.env.production` 后的标准重启

```bash
set -a; . ./.env.production; set +a
npm run build && pm2 reload repairkit
```
> 凡是改了 `NEXT_PUBLIC_*` 变量，必须 `npm run build`（编译期注入）；只改后端变量可只 `pm2 reload`。
