> 现状：服务器已在阿里云新加坡 ECS（IP `47.84.33.52`）跑起来了，用 Nginx + PM2（应用名 `repairkit`，端口 4173）+ 自建 PostgreSQL。代码在 GitHub `tongyanzhanggithub/RK`，最新改动**尚未** `git pull` 到服务器。
> 海外服务器：**网站和邮箱都不需要 ICP 备案**，只需域名实名认证。

## 阶段 0 · 先把最新代码推到服务器（不依赖域名，随时可做）

SSH 进服务器，进项目目录，整段执行：

```bash
git pull
set -a; . ./.env.production; set +a
npx prisma generate
npx prisma db push
node scripts/seed-hero.js
npm run build
pm2 reload repairkit
```

- [ ] 跑完无报错，浏览器开 `http://47.84.33.52` 能看到新配色 + 移动端汉堡菜单
- [ ] 进 `http://47.84.33.52/admin/login` 后台是干净独立外壳（无前台页眉）

> 易错点：① 一定先 `set -a; . ./.env.production; set +a` 再 `db push`，否则连不上库；② 别漏 `prisma db push`，否则 build 找不到新表报错。

## 阶段 3 · HTTPS（免费证书 + 自动续期）

服务器上（Nginx 已装）：

```bash
sudo apt update && sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d partavio.com -d www.partavio.com
```

按提示填邮箱、同意条款、选择「自动把 http 跳转到 https」。

- [ ] certbot 成功签发证书
- [ ] `https://partavio.com` 出现小绿锁
- [ ] `http://partavio.com` 自动跳转到 `https://`
- [ ] certbot 自动续期已就绪（`sudo certbot renew --dry-run` 通过）

### ⚠️ 上 HTTPS 后必须改一处环境变量
编辑服务器 `.env.production`，把站点地址改成 https 域名（登录 Cookie 的 secure 开关依赖它）：

```
NEXT_PUBLIC_SITE_URL="https://partavio.com"
```

然后重建并重启：

```bash
set -a; . ./.env.production; set +a
npm run build && pm2 reload repairkit
```

- [ ] `NEXT_PUBLIC_SITE_URL` 已改为 `https://partavio.com`
- [ ] 重建重启后，后台能正常登录（不再出现「点登录又跳回登录页」）

## 阶段 5 · 收款接入

### 5A · Airwallex（主刷卡通道）
- [ ] 开通 Airwallex 商户账户（用营业执照）
- [ ] 拿到 API 凭据，填进 `.env.production`：
  ```
  AIRWALLEX_CLIENT_ID="..."
  AIRWALLEX_API_KEY="..."
  AIRWALLEX_ENV="demo"      # 先 demo 测试，正式改 prod
  ```
- [ ] 重建重启 → 购物车出现「信用卡」按钮（配了 Airwallex 会自动取代 Stripe）
- [ ] demo 环境下完整走一笔测试支付
- [ ] 正式开张前把 `AIRWALLEX_ENV` 改为 `prod`

### 5B · PayPal（备用钱包，可选）
- [ ] PayPal 开发者后台建 REST App，拿 Client ID / Secret
- [ ] 填进 `.env.production`：
  ```
  PAYPAL_CLIENT_ID="..."
  PAYPAL_SECRET="..."
  PAYPAL_ENV="sandbox"      # 测试 sandbox，正式 live
  PAYPAL_WEBHOOK_ID="..."   # 后台建 Webhook 后获得，用于校验回调
  ```
- [ ] 重建重启 → 购物车出现「PayPal」按钮
- [ ] sandbox 走一笔测试，再切 live

> 不配支付也能上线：购物车暂时只引导走 WhatsApp 询价；批发本就走电汇。

## 阶段 6 · 发信邮箱（找回密码/订单/发货/弃单通知）

- [ ] 选一家事务邮件服务（推荐 Resend / Brevo，详见之前对比）
- [ ] 填进 `.env.production`（以 Resend 为例）：
  ```
  SMTP_HOST="smtp.resend.com"
  SMTP_PORT="465"
  SMTP_SECURE="true"
  SMTP_USER="resend"
  SMTP_PASS="re_xxx"                       # API Key
  SMTP_FROM="Partavio <orders@partavio.com>"
  ADMIN_NOTIFY_EMAIL="admin@partavio.com"  # 收到的新订单/询盘通知
  ```
- [ ] （自定义发件域名要在服务商加 SPF/DKIM 解析记录并验证通过）
- [ ] 重建重启 → 后台触发一封找回密码邮件，确认能收到

## 通用：改完 `.env.production` 后的标准重启

```bash
set -a; . ./.env.production; set +a
npm run build && pm2 reload repairkit
```
> 凡是改了 `NEXT_PUBLIC_*` 变量，必须 `npm run build`（编译期注入）；只改后端变量可只 `pm2 reload`。
