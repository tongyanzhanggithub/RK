# 阿里云部署指南 — RepairKit Supply

面向海外客户（英国/欧洲/中东/东南亚）的 Next.js 14 + PostgreSQL + Stripe 独立站，部署到**阿里云 ECS**。

---

## 0. 架构总览

```
用户 ──► Cloudflare/阿里云CDN(可选,强烈建议) ──► 阿里云ECS(Nginx :80/443)
                                                      │
                                                      ▼
                                              Next.js (PM2, 127.0.0.1:4173)
                                                      │
                                                      ▼
                                          阿里云 RDS PostgreSQL (内网)
图片：ECS 本机磁盘 public/uploads/（单机够用）→ 上量后迁 OSS
支付：Stripe（海外，需 ECS 能直连 Stripe → 必须用海外节点）
```

> ⚠️ **务必用海外节点（新加坡/法兰克福/迪拜），不要用国内节点**：① 国内节点要 ICP 备案；② 国内访问 Stripe 不稳；③ 海外客户访问国内机房慢。

---

## 1. 买资源（阿里云国际版 alibabacloud.com）

| 资源 | 规格建议（起步） | 说明 |
|---|---|---|
| **ECS 云服务器** | 2核4G，Ubuntu 22.04，**新加坡 ap-southeast-1** | 系统盘 40G+ ESSD；分配公网 IP |
| **RDS PostgreSQL** | 1核2G 起，**与 ECS 同地域同 VPC** | 初期想省钱可跳过，先在 ECS 本机装 PG（见 3-B） |
| **OSS（可选）** | 同地域 Bucket | 图片走 CDN 时才需要，初期可不买 |
| 域名 + 证书 | — | 域名可在任意注册商买；HTTPS 用 Let's Encrypt 免费证书 |

**安全组**（ECS 控制台 → 安全组）：入方向放行 **80、443、22**。22 建议只放行你自己的公网 IP。

## 2. 初始化服务器

SSH 登录 ECS 后：

```bash
# 拉代码（或先 git clone 你的仓库到 /var/www/repairkit）
sudo mkdir -p /var/www && cd /var/www
sudo git clone https://github.com/tongyanzhanggithub/RK.git repairkit
cd repairkit

# 一键装环境（Node 20 / Nginx / PM2 / Certbot）
bash deploy/setup-ecs.sh
```

## 3. 准备数据库

### 3-A 用 RDS PostgreSQL（推荐）
1. RDS 控制台创建账号和数据库 `rk`。
2. 把 ECS 的内网 IP 加入 RDS 白名单（同 VPC 走内网）。
3. 复制**内网连接地址**，填进下一步的 `DATABASE_URL`。

### 3-B 省钱方案：ECS 本机自装 PostgreSQL
```bash
sudo apt-get install -y postgresql
sudo -u postgres psql -c "CREATE USER rk WITH PASSWORD '改成强密码';"
sudo -u postgres psql -c "CREATE DATABASE rk OWNER rk;"
# DATABASE_URL = postgresql://rk:改成强密码@localhost:5432/rk?schema=public
```
> 自装方案：务必配置 `pg_dump` 定时备份（见第 8 节）。RDS 自带自动备份，更省心。

## 4. 配置环境变量

```bash
cd /var/www/repairkit
cp .env.production.example .env.production
nano .env.production   # 按注释逐项填：DATABASE_URL / 域名 / Stripe live key / 会话密钥 / SMTP ...
```

生成会话密钥：
```bash
node -e "console.log(require('crypto').randomBytes(48).toString('base64url'))"
```
`ADMIN_SESSION_SECRET` 和 `CUSTOMER_SESSION_SECRET` 各生成一个不同的值。

## 5. 建库、构建、启动

```bash
cd /var/www/repairkit
npm ci                                   # 装依赖
export $(grep -v '^#' .env.production | xargs)   # 临时加载变量供下面命令使用

npx prisma generate                      # 生成 Prisma Client
npx prisma db push                       # 在 RDS 上建表（首次部署）
npm run db:seed                          # 写入产品目录种子数据（可选；线上不想要演示订单可后续在后台删）
node scripts/seed-taxonomy.js            # 写入前台分类导航 + 维修指南（从产品派生，幂等）
ADMIN_EMAIL='admin@your-domain.com' ADMIN_PASSWORD='强密码' npm run admin:create   # 建管理员

npm run build                            # 生产构建

# 用 PM2 守护启动
pm2 start deploy/ecosystem.config.js
pm2 save                                 # 保存进程列表，开机自启
pm2 status
```

此时本机 `curl http://127.0.0.1:4173` 应返回页面。

> **更新发布**（以后每次改完代码）：
> ```bash
> cd /var/www/repairkit && git pull
> npm ci && npx prisma generate && npx prisma db push && npm run build
> pm2 reload repairkit
> ```

## 6. Nginx + HTTPS

```bash
# 1) 配反向代理
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/repairkit
sudo nano /etc/nginx/sites-available/repairkit     # 把 your-domain.com 改成你的域名
sudo ln -s /etc/nginx/sites-available/repairkit /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx

# 2) 先把域名 A 记录解析到 ECS 公网 IP（在域名 DNS 处配置），生效后：
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
# certbot 自动签发证书 + 配置 443 + http→https 跳转 + 自动续期
```

## 7. Stripe 生产配置

1. Stripe Dashboard 切到 **Live** 模式，取 `sk_live_...` 填 `STRIPE_SECRET_KEY`。
2. Developers → Webhooks → Add endpoint：
   - URL：`https://你的域名/api/stripe/webhook`
   - 事件：`checkout.session.*`、`payment_intent.*`、`charge.succeeded`、`charge.refunded`、`charge.dispute.*`
3. 取该 endpoint 的 `whsec_...` 填 `STRIPE_WEBHOOK_SECRET`，`pm2 reload repairkit` 生效。
4. 在网站下一笔真实小额订单，确认后台订单状态从 PENDING→PAID（webhook 通了）。

## 8. 备份与运维（带交易/库存的站，必做）

- **数据库备份**：RDS 开「自动备份」；自装 PG 则加 cron：
  ```bash
  # 每天 03:00 备份并保留 14 天
  0 3 * * * pg_dump "$DATABASE_URL" | gzip > /var/backups/rk_$(date +\%F).sql.gz; find /var/backups -name 'rk_*.sql.gz' -mtime +14 -delete
  ```
- **图片备份**：`public/uploads/` 跟着 ECS 磁盘走 —— 给云盘开「自动快照」，或定期同步到 OSS。
- **日志**：`pm2 logs repairkit`；Nginx 日志在 `/var/log/nginx/`。
- **监控**：阿里云云监控配 CPU/内存/磁盘告警。

---

## 9. 抗流量（按量级）

| 流量 | 做法 |
|---|---|
| 试销 | 单台 2核4G + 本机或小 RDS，**前面挂 Cloudflare（免费 CDN+防DDoS+缓存）** |
| 稳定出单 | ECS 2核4G + 独立 RDS（1核2G→2核4G）；图片迁 OSS+CDN |
| 促销爆量 | 2 台 ECS + **ALB 负载均衡**；RDS 开连接池；静态全量走 CDN。注意：多实例前必须先把图片上传迁到 OSS（本机磁盘不共享），并可把 PM2 设为 cluster 模式 |

> 自动按 IP 切币种/运费/税：需在 ECS 前挂 Cloudflare（透传 `cf-ipcountry`，Nginx 配置已留好）。不挂 CDN 时该功能降级为「手动国家切换器」，不影响下单。

## 10. 本地开发（迁到 Postgres 后）

本地不再用 SQLite，改用 Docker 起一个 Postgres（与线上同引擎）：
```bash
docker compose -f docker-compose.dev.yml up -d
cp .env.example .env          # 默认 DATABASE_URL 已指向本地 5432
npx prisma db push
npm run db:seed
ADMIN_EMAIL=admin@example.com ADMIN_PASSWORD=ChangeMe123 npm run admin:create
npm run dev                   # http://127.0.0.1:4173
```
没装 Docker 也可以：本机装 PostgreSQL，或直接把 `DATABASE_URL` 指向一个云上的开发库。
